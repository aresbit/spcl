# SPCL 中文使用指南

## 1. SPCL 是什么

SPCL 是一个基于 C 实现的配置解释器，遵循 CCL 核心语法：

- `key = value`
- 缩进表示“值的续行/嵌套”

你可以把它理解为：

1. 输入一段配置文本
2. 解释器把文本解析成一棵树状模型
3. 输出整棵树，或者查询某条路径的子树

---

## 2. 输入是什么

输入可以来自文件，也可以来自标准输入（stdin）。

典型输入示例（`examples/example.spcl`）：

```txt
/= CCL document
title = CCL Example

database =
  enabled = true
  ports =
    = 8000
    = 8001
    = 8002
```

语法要点：

- 基础语法是 `key = value`
- 缩进表示“值的续行/嵌套”

---

## 3. 解释器做了什么

解释流程（从输入到输出）如下：

1. 第一阶段：按行扫描，解析为键值对序列（`key`, `value`）
2. 第二阶段：对每个 `value` 再递归解析
3. 构建统一树模型（同名 key 会合并）
4. 按需求输出整棵树，或输出某个查询路径

可以理解为：

- 文本输入 -> `parse` -> 中间键值对
- 中间键值对 -> `decode/fix` -> 树模型
- 树模型 -> `pretty/query` -> 终端输出

---

## 4. 输出是什么

输出是“规范化后的树形文本”。

### 4.1 输出整棵树

```sh
./build/bin/cclq examples/example.spcl
```

解释器会打印整个模型的树形结构。

### 4.2 查询子路径

```sh
./build/bin/cclq examples/example.spcl database=ports
```

含义：

- 查询路径 `database -> ports`
- 只输出这个子树

## 5. 常用命令

构建：

```sh
make all
```

测试：

```sh
make test
```

帮助：

```sh
./build/bin/cclq --help
```

stdin 输入示例：

```sh
cat examples/example.spcl | ./build/bin/cclq --
```

---

## 6. 失败与错误

常见错误：

- 语法错误（缺少 `=`、缩进层级不合法）
- 查询路径不存在（`Query not found`）
- 文件无法打开

排查建议：

1. 先运行 `./build/bin/cclq <file>` 看整体解析是否正常
2. 再运行具体查询 `a=b=c`
3. 检查缩进是否一致（建议空格，不要混 tab）

---

## 7. 与原 CCL 的关系

- 对齐 CCL 的核心配置写法
- 不再支持 `SKILLS:` / `PROMPT:` 冒号块扩展
